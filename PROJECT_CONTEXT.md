# PROJECT_CONTEXT

Проект: BUILD_TRACKER

Описание:
Flutter-приложение для контроля строительства / BuildApp.

Важные правила:
- Не ломать существующую бизнес-логику
- Перед массовыми изменениями показывать diff
- Не менять API endpoints без подтверждения
- Аккуратно работать с авторизацией, токенами и ролями
- Проверять связь с backend Node.js
- UI менять аккуратно, чтобы не сломать текущий дизайн

Технологии:
- Flutter
- Dart
- Android / iOS / Web
- REST API
- JWT / авторизация
- Node.js backend

Опасные места:
- Авторизация
- Роли admin / исполнитель
- Объекты строительства
- Задачи
- Уведомления
- API-запросы
